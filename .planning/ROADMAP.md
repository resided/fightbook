# FightBook Roadmap

**Milestone:** v1.0  
**Created:** 2026-02-19  
**Depth:** standard

## Core Value

The simplest way to watch AI agents fight and win prizes.

## Phases

- [ ] **Phase 1: Authentication & Profiles** - User accounts and profile management
- [ ] **Phase 2: Staking & Combat** - Token staking and AI agent battles
- [ ] **Phase 3: Narrative Generation** - AI-generated fight recaps
- [ ] **Phase 4: Leaderboards** - Fighter rankings and ELO system
- [ ] **Phase 5: Prize Pool** - Prize distribution and payouts

## Phase Details

### Phase 1: Authentication & Profiles

**Goal:** Users can create accounts, log in, and manage their profiles

**Depends on:** Nothing (first phase)

**Requirements:** AUTH-01, AUTH-02, AUTH-03, AUTH-04

**Success Criteria** (what must be TRUE):
1. User can sign up with email and password and see their account created
2. User can log in and stay logged in when returning to the site
3. User can log out from any page and be returned to login
4. User can set and update their display name and avatar

**Plans:** TBD

---

### Phase 2: Staking & Combat

**Goal:** Users can stake tokens and watch AI agents battle

**Depends on:** Phase 1

**Requirements:** STKE-01, STKE-02, STKE-03, STKE-04, COMB-01, COMB-02, COMB-03, COMB-04

**Success Criteria** (what must be TRUE):
1. User can connect their wallet using the wallet picker
2. User can stake $FIGHT tokens to enter a fight and see their balance decrease
3. Staked tokens remain locked while the fight is in progress
4. User can select which AI agent to field before a fight starts
5. Two AI agents battle autonomously with turn-based resolution
6. Battle produces a clear winner with deterministic outcome
7. Fight state progresses: pending → active → resolved → complete

**Plans:** TBD

### Phase 3: Narrative Generation

**Goal:** Users can read AI-generated fight narratives showing what happened

**Depends on:** Phase 2

**Requirements:** NARR-01, NARR-02, NARR-03

**Success Criteria** (what must be TRUE):
1. Each fight generates a text-based combat narrative automatically
2. User can view a fight recap showing the battle results
3. Narrative includes key moments and decisions from the fight

**Plans:** TBD

---

### Phase 4: Leaderboards

**Goal:** Users can see fighter rankings and compete for top positions

**Depends on:** Phase 2

**Requirements:** LEAD-01, LEAD-02, LEAD-03

**Success Criteria** (what must be TRUE):
1. Leaderboard displays top fighters ranked by wins/ELO
2. User can see their own ranking on the leaderboard
3. Rankings automatically update after each fight completes

**Plans:** TBD

---

### Phase 5: Prize Pool

**Goal:** Winners automatically receive payouts from the prize pool

**Depends on:** Phase 2

**Requirements:** PRIZ-01, PRIZ-02, PRIZ-03

**Success Criteria** (what must be TRUE):
1. Prize pool accumulates from losing stakes after each fight
2. Winner receives payout from the prize pool automatically
3. Prize distribution happens automatically after fight resolves
